# TeachSack 设计上下文（Design Context）

> 本文件是 impeccable 技能在 TeachSack（教学工具箱）项目中的唯一权威设计上下文。
> 任何 UI / 视觉 / 交互 / 文案调整，必须先遵循本文件，再套用技能通用原则。
> 核心防线：**克制、去 AI 味、线条化、双主题一致性**。

---

## 1. 产品概述（Product）

- **产品名**：TeachSack 教学工具箱（对外品牌，内部曾用名"教学工具箱"）
- **形态**：Electron 单文件应用（`教学工具箱.html`），兼顾桌面客户端与官网落地页（`index.html`）
- **当前版本**：v1.2.14（发版节奏稳定，改动走本地打包 → Gitee releases → 自动更新）
- **分发**：Gitee `qwezasxf/teaching-toolbox`，autoUpdater 源 `releases/download/latest/`

## 2. 目标用户（Target Audience）

- 一线教师、培训讲师、教研人员，非技术背景为主
- 使用场景：课堂互动、出题组卷、知识闯关、即时反馈收集
- 设备：Windows 桌面为主，屏幕分辨率跨度大（笔记本 → 大屏投影）
- 审美预期：**清爽、专业、不花哨**——比"酷炫"更看重"看得清、用得顺"

## 3. 核心用例（Use Cases）

1. 主界面工具卡入口 → 各功能页（知识闯关 / 自定义题目 / 设置）
2. 浅色 / 深色双主题切换，且两主题下文字、卡片、边框都必须**清晰可读**
3. 弹窗体系：退出确认、激活码、反馈、更新日志——统一用 `showCustomModal`
4. 性能：主界面卡片多、长列表（自定义题目 / 更新日志）需屏外跳过渲染

## 4. 品牌调性（Brand Personality / Tone）

| 维度 | 取值 |
|------|------|
| 整体气质 | 克制、专业、安静；像一件趁手的教具，而非炫技 Demo |
| 情绪 | 稳定、可信赖、低认知负载 |
|  Differentiation（记忆点） | **线条感**——统一线性图标 + 留白节奏，而非色块与发光 |
| 绝对禁忌 | 暴露"AI 生成"痕迹（emoji 堆砌、机械罗列、生成腔文案） |

## 5. 设计原则（DO / DON'T）

### ✅ DO

- **去 AI 味**：更新日志 / 提示文案用短句、口语化、不罗列；不用 emoji 当装饰
- **线条图标**：功能图标统一用内联 SVG 线条（`stroke` 描边），不混用 emoji
- **双主题变量**：所有颜色走 CSS 变量（`--card-bg` / `--text-primary` / `--text-secondary` / `--text-muted` / `--border-color` / `--shadow-color`），禁止硬编码颜色
- **浅色适配必查**：新增任何组件，必须同时在深色 + 浅色下验证文字 / 边框 / 背景对比
- **动效克制**：保留现有入场动画（如 `cardRise`），新动效只用状态过渡（ease-out），不回弹
- **性能隔离**：长列表 / 大容器用 `content-visibility:auto` + `contain-intrinsic-size`；独立卡片用 `contain:layout style paint`

### ❌ DON'T

- 用 emoji（📱🎯 等）充当功能图标或装饰——会导致尺寸不一致、浅色下发灰
- 在弹窗 / 日志里写"我们优化了…""新增了 AI 能力…"这类生成腔
- 硬编码 `#fff` / `#000` / 渐变发光等"AI 默认审美"
- 改动任何功能代码来配合视觉调整（历史教训：上一个 AI 常为了视觉动功能，严禁）
- 让 `.toolbox-header` 等头部容器重播入场动画导致图标闪偏移
- 任何破坏自动更新通路的改动（latest.yml / main.js update 源）

## 6. 主题 Token（Theme Tokens）

```css
:root { /* 深色（默认） */
  --card-bg: #1e2330;
  --text-primary: #f5f7fa;
  --text-secondary: #c2c8d4;
  --text-muted: #8a93a6;
  --border-color: #2e3444;
  --shadow-color: rgba(0,0,0,.35);
}
[data-theme="light"] { /* 浅色 */
  --card-bg: #ffffff;
  --text-primary: #1a1d24;
  --text-secondary: #4a5160;
  --text-muted: #8a93a6;
  --border-color: #e3e6ee;
  --shadow-color: rgba(20,30,60,.10);
}
```

> 新增组件取色一律引用上述变量；浅色模式文字最低对比需满足 `--text-muted` 在 `--card-bg` 上仍可读。

## 7. 图标规范（Icon Convention）

- 统一内联 SVG，`viewBox="0 0 24 24"`，`stroke="currentColor"` `stroke-width="1.8"` `fill="none"`
- 全屏按钮等"双态"图标：两个状态都给线条 SVG，避免 emoji 尺寸跳变
- 设置齿轮、功能卡图标：维持现有线性风格，不替换成彩色 / 实心图标

## 8. 弹窗规范（Modal）

- 统一入口 `showCustomModal({ title, content, icon, showCancel, cancelText, ... })`
- 退出确认：`icon:'none'`（不显示老图标）
- 激活码弹窗：提供 `showCancel:true, cancelText:'关闭'`
- 文案短、动作清晰；不堆砌说明

## 9. 性能约束（Performance）

- 纯 CSS containment，不引入运行时库、不改功能逻辑
- 屏外跳过渲染：`content-visibility:auto; contain-intrinsic-size:auto 600px~800px`
- 重绘隔离：`.tool-card / .quiz-card / .custom-modal / .feedback-modal { contain:layout style paint }`

## 10. 协作约定（Workflow）

- 单文件应用的小修复 / 视觉调优：**直接由主助手落地**，不走多智能体 / 子 agent 流程
- 每次视觉改动后：本地打包 → 上传 Gitee → 远程 `sha512` 逐字核验 → 确认自动更新通路正常
- 改动前先 `git status` 确认源码确有变更，避免无效重传
