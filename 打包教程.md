# 🧰 教学工具箱 - Electron 打包教程

本教程将指导你将教学工具箱打包成可直接安装的 Windows 桌面应用（.exe 文件）。

---

## 📋 打包前准备

### 1. 确保项目结构完整

```
D:\jxgj\
├── 教学工具箱.html      ← 主应用界面
├── package.json         ← 项目配置（已配置好）
├── main.js              ← Electron 主进程
├── preload.js           ← 安全预加载脚本
├── server.js            ← 备用HTTP服务器
├── sounds/              ← 音效文件夹
│   ├── card_flip.mp3.wav
│   └── card_fly.mp3.wav
└── node_modules/        ← 依赖文件夹
```

### 2. 确认 package.json 配置正确

打开 `package.json`，确保包含以下内容：

```json
{
  "name": "teaching-toolbox",
  "version": "1.0.0",
  "main": "main.js",
  "scripts": {
    "start": "node server.js",
    "start:electron": "electron .",
    "build:win": "electron-builder --win"
  },
  "build": {
    "appId": "com.teaching.toolbox",
    "productName": "教学工具箱",
    "win": {
      "target": [
        { "target": "nsis", "arch": ["x64", "ia32"] },
        { "target": "portable", "arch": ["x64"] }
      ]
    }
  },
  "devDependencies": {
    "electron": "^28.2.0",
    "electron-builder": "^24.9.1"
  }
}
```

## 🚀 开始打包

### 步骤1：删除不完整的 Electron

如果之前安装过但不完整，先删除：

```bash
rmdir /s /q node_modules\electron
```

### 步骤2：配置国内镜像（关键！）

```bash
set ELECTRON_MIRROR=https://npmmirror.com/mirrors/electron/
set ELECTRON_BUILDER_BINARIES_MIRROR=https://npmmirror.com/mirrors/electron-builder-binaries/
```

### 步骤3：重新安装依赖

```bash
npm install
```

### 步骤4：测试 Electron 是否能正常运行

```bash
npm run start:electron
```

> 如果弹出教学工具箱窗口，说明 Electron 安装成功！

### 步骤5：执行打包命令

```bash
npm run build:win
```

## ⚠️ 常见问题解决

### 问题1：Electron 安装失败

**原因**：网络问题导致下载不完整

**解决方案**：
```bash
# 1. 删除不完整的安装
rmdir /s /q node_modules\electron

# 2. 配置镜像
set ELECTRON_MIRROR=https://npmmirror.com/mirrors/electron/

# 3. 重新安装
npm install electron@28.2.0
```

### 问题2：打包时提示缺少图标

**原因**：配置中指定了图标文件但不存在

**解决方案**：
- 删除 `package.json` 中的 `"icon": "icon.ico"` 配置
- 或准备一个 256x256 的 icon.ico 文件放在项目根目录

### 问题3：打包速度很慢

**原因**：electron-builder 需要下载依赖

**解决方案**：
```bash
# 设置国内镜像
set ELECTRON_BUILDER_BINARIES_MIRROR=https://npmmirror.com/mirrors/electron-builder-binaries/
```

### 问题4：打包后无法运行

**原因**：可能缺少必要的文件

**解决方案**：
- 检查 `package.json` 中的 `build.files` 配置
- 确保包含所有必要文件：
  ```json
  "files": [
    "main.js",
    "preload.js",
    "教学工具箱.html",
    "sounds/**/*"
  ]
  ```
