## 🔴 官网部署卡死根因（重大排查，已解决）
- 现象：下载链接/文案/SEO/截图 四次推送（eea9f07/249697a/ba28260 + .nojekyll cf372ae）后，线上 `teachsack.cn` 仍停留在 7/10 旧版；`curl -I` 显示 `Last-Modified: Fri, 10 Jul` 且 `Cache-Control: max-age=600`。`git ls-remote` 确认远程 main 已是最新提交 → 说明 **GitHub Pages 构建根本没跑（一直保留最后一个成功构建 7/10）**。

- 修复：`git rm --cached server-deploy`（从索引摘掉 gitlink，**保留本地文件夹**，不影响 Cloudflare API 的 wrangler 部署）→ 提交 6de0e8b → `git push origin main`（cf372ae..6de0e8b）。推送后约 20s 线上 `Last-Modified` 变今天，gitee/og:title/features-screenshot-img 全部命中、旧文案消失 → **官网完全更新成功**。
- 教训：GitHub Pages 站点若推送后长期不更新且 Last-Modified 不变，**先查仓库里有无悬空子模块 gitlink（`git ls-tree main` 看 160000 条目）与 .gitmodules 是否齐全**，再查 Jekyll/源分支。另：链式 `git add && git commit && git push` 在本仓会出现误导性 "no changes added" 报错（commit 其实成功了），应分开执行并单独看 push 结果。
